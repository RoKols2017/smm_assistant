# Security

## Основные правила

| Нельзя | Нужно |
|--------|-------|
| Коммитить `.env` | Хранить секреты в переменных окружения |
| Логировать токены | Логировать только технический контекст ошибок |
| Передавать секреты через URL | Использовать заголовки или конфиг окружения |
| Держать старые ключи без ротации | Регулярно ротировать API-ключи |

## Рекомендуемый `.env`

```bash
OPENAI_API_KEY=sk-your-actual-key-here
VK_TOKEN=your-actual-vk-token-here
VK_GROUP_ID=123456789
TG_TOKEN=1234567890:your-actual-bot-token-here
TG_CHAT_ID=@your_channel_or_chat_id
LOG_LEVEL=INFO
MAX_RETRIES=3
TIMEOUT=30
```

## Перед запуском

```bash
python test_env.py
```

Проверяйте:

- наличие обязательных переменных;
- корректность токенов;
- доступность внешних сервисов;
- отсутствие секретов в логах и коде.

## Практики для интеграций

### OpenAI

- следите за лимитами и стоимостью запросов;
- не отправляйте в модель чувствительные данные без необходимости.

### VK

- используйте токен с нужными правами (`wall`, `photos`, `groups`);
- учитывайте лимиты API и ограничения на загрузку изображений.

### Telegram

- храните bot token вне репозитория;
- проверяйте права бота в канале или чате перед публикацией.

## Если ключи скомпрометированы

1. Отзовите ключи и токены.
2. Выпустите новые значения.
3. Проверьте логи на подозрительную активность.
4. Обновите конфигурацию окружения.

Полная исходная памятка пока сохранена в корневом `SECURITY.md`.
